# Boss DOM 适配维护

扩展只读取用户当前已打开的岗位详情 DOM，不调用 Boss 私有 API。

## 页面支持

- 独立岗位详情页。
- 左侧岗位列表、右侧岗位详情的分栏页。

## 分栏页身份与公司名

- 分栏页地址栏在切换岗位后可能保持不变，禁止把列表 URL 直接当作岗位唯一键。
- 优先读取当前岗位卡片的 `data-jobid`、详情链接或链接内 `jobId`；同一岗位更新，
  不同岗位必须追加到列表。
- 详情区没有公司名时，从与详情标题一致的当前岗位卡片补取公司名。
- Boss 分栏页优先从右侧面板 `.boss-info-attr` 的 `公司名 · HR 职位` 中提取
  公司名，再使用详情公司元素和当前岗位卡片兜底。
- Boss 薪资数字使用 `kanzhun-Regular` PUA 字符：`U+E031` 到 `U+E03A`
  必须在写入存储前解码为 `0` 到 `9`；不得把 PUA 原文导出到 JSON。
- 修改这些规则后，至少用同一个搜索页连续保存两个岗位，确认弹窗数量为 2，
  且两条记录均显示各自公司名。

## 改版排查

当页面提示“未识别到岗位名称”或“未识别到岗位正文”时：

1. 确认岗位详情已完整加载。
2. 在开发者工具中定位岗位名称、公司、薪资、地点和正文元素。
3. 详情字段选择器修改 `lib/boss-extractor.mjs`；当前岗位卡片 DOM 适配修改
   `content/content.js` 中的 `activeJobCard()`。
4. 为新结构增加 `tests/boss-extractor.test.mjs` 夹具。
5. 运行 `npm test`。
6. 在独立详情页和分栏页各手动保存一次。
7. 导出 JSON 并使用 Resume-Tailor 导入验证。

不要：

- 读取 Cookie 或本地登录凭据；
- 逆向或调用平台私有 API；
- 自动翻页、批量点击或自动投递；
- 用宽泛的 `body.innerText` 代替岗位详情选择器；
- 把推荐岗位、聊天或广告混入岗位正文。
- 把 Boss 薪资 PUA 字符原样保存，导致弹窗和 JSON 出现方块乱码。
