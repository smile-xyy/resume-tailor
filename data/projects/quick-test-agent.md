# 新能源汽车售后故障初筛与工单协同 Multi-Agent 系统

## 背景

系统面向车主 App 和售后服务顾问工作台，处理车辆故障咨询、维保预约、
保修政策查询、服务建议、工单创建和人工升级。

## 流程

1. 接收车主的故障描述和车辆信息。
2. 识别用户意图并提取故障槽位。
3. 查询车辆档案、历史维保记录和保修政策。
4. 从维修手册、常见故障 FAQ 和历史工单中检索依据。
5. 生成故障初筛结果和服务建议。
6. 信息不足时继续多轮询问并通过 Redis 保存会话状态。
7. 需要人工处理时生成工单摘要并转交售后顾问。
8. 使用 OpenTelemetry Trace 记录执行链路。

## 技术与边界

- 使用 LangGraph 和 FastAPI 组织多智能体协作链路。
- 使用统一 ToolSchema 封装业务工具。
- 使用 RAG、Query 改写、向量检索和重排序提供可引用依据。
- 使用 AsyncIO 并行执行车辆查询、知识检索和工单校验。
- 使用超时控制、异常隔离和降级回复处理局部服务异常。

## 已有结果

- 支持复杂售后问题的连续处理和推荐依据追溯。
- 需要时自动生成工单摘要并转交人工。
- 端到端响应时间降低约 30%。
