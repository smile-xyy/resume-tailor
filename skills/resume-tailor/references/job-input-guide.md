# 岗位信息导入用户指南

## 使用时机

进入岗位导入阶段时，先确定项目根目录的绝对路径，再把下方说明直接展示给用户。
不要只回复“上传 JSON、截图或粘贴 JD”，也不要只让用户去看 README。

首次进入该阶段时完整展示三种方式；用户已经明确选择方式后，只展示该方式的
后续步骤和故障处理。整条回复最后只保留一个行动问题。

## 面向用户的首次引导

请提供你想分析的岗位信息。推荐使用 Chrome 插件，尤其适合一次收集多个岗位；
也可以直接发截图或粘贴完整 JD。

### 方式一：Chrome 岗位摘录器（推荐）

支持 Boss 直聘、前程无忧、智联招聘和猎聘。插件只保存你主动点击的岗位，
不会自动投递，也不会读取 Cookie。

#### 第一次安装（只需操作一次）

1. 先复制插件文件夹路径：`<项目根目录绝对路径>/browser-extension/`。
2. 在 Chrome 地址栏输入 `chrome://extensions` 并回车。
3. 打开页面右上角的“开发者模式”。
4. 点击左上角“加载未打包的扩展程序”。
5. 在文件选择窗口中选择刚才的 `browser-extension` 文件夹。要选择整个文件夹，
   不要选择其中的 `manifest.json`，也不要选择项目上一级目录。
6. 页面出现“岗位摘录器”卡片即表示安装成功。
7. 点击 Chrome 右上角拼图图标，在“岗位摘录器”旁点击图钉，将它固定到工具栏。

如果 Chrome 提示清单文件缺失，通常是选错了目录；重新选择包含
`manifest.json` 的 `browser-extension` 文件夹。

如果此前已经安装过旧版，不用卸载：更新项目文件后进入 `chrome://extensions`，
点击“岗位摘录器”卡片上的“重新加载”，再刷新已打开的招聘页面。旧版留下的
“公司待确认”岗位可在对应岗位页重新保存一次，插件会原位补全，不会重复新增。
旧版已经保存的方块薪资会在打开新版弹窗时自动解码；公司名仍需在对应岗位页
重新保存一次才能从当前右侧岗位面板补全。

#### 收集并导出岗位

1. 在支持的招聘平台打开一个具体岗位的详情页，不要停留在岗位搜索列表页。
2. 等岗位名称、公司、职责和任职要求加载完整；有“展开”时先展开全文。
3. 点击页面右侧的“保存当前岗位 · 0”（数字是当前已收集数量）；看到按钮上方
   跳出“已添加岗位”或“已更新岗位”即保存成功。
4. 继续打开其他岗位详情并重复保存；不同岗位会追加，同一岗位再次保存才会更新，
   一个导出文件可以包含多个岗位。
5. 点击 Chrome 工具栏中的“岗位摘录器”图标，确认岗位数量和名称。
6. 点击“导出 jobs.json”。浏览器会下载类似
   `resume-tailor-jobs-2026-07-12T10-30-00.jobs.json` 的文件。
7. 把这个 `.jobs.json` 文件直接拖进当前会话。我会自动导入、去重并检查字段。

#### 插件没有识别到岗位时

- 确认打开的是岗位详情页，不是搜索结果页、公司主页或登录页；
- 展开完整 JD，刷新页面后再点一次“保存当前岗位”；
- 如果扩展刚更新，回到 `chrome://extensions`，在“岗位摘录器”卡片上点击
  “重新加载”，然后刷新招聘页面；
- 仍然失败时不要反复尝试，改用下方截图或粘贴 JD。系统不会根据空字段猜测岗位。

### 方式二：岗位截图

1. 打开岗位详情页并展开职责、任职要求。
2. 优先截一张完整长截图；内容太长时可连续截多张，并说明这些截图属于同一岗位。
3. 截图至少要清楚包含：职位名称，以及“岗位职责”或“任职要求”正文；最好同时
   包含公司、地点、薪资、经验、学历和岗位链接。
4. 一次可以上传多个岗位，但请让每个岗位的截图容易分组。

如果职位名称或职责/要求无法识别，我会指出缺少的部分，并请你重新截图或改用
插件/粘贴方式；不会把推荐岗位、广告或模糊文字当作 JD。

### 方式三：直接复制 JD

从岗位详情页复制正文并按下面格式粘贴；没有的信息可写“未知”：

```text
职位名称：
公司名称：
岗位职责：
任职要求：
经验/学历：
薪资/地点（可选）：
岗位链接（可选）：
```

至少要有“职位名称”，并且“岗位职责/任职要求”至少有一项。只有公司名、薪资或
职位标签不足以进行可靠匹配。

回复末尾使用这一个行动问题：

> 请现在直接发送插件导出的 `.jobs.json`、岗位截图或完整 JD；如果想让我陪你一步步安装插件，请回复“安装插件”。

## 用户回复“安装插件”后的逐步陪同

一次只让用户完成一个步骤，并等待确认：

1. 给出当前项目 `browser-extension/` 的绝对路径，请用户复制，完成后回复“已复制”。
2. 请用户打开 `chrome://extensions` 和“开发者模式”，完成后回复“已打开”。
3. 请用户点击“加载未打包的扩展程序”并选择该目录，完成后回复安装结果或发截图。
4. 安装成功后，请用户固定扩展，再打开一个岗位详情页。
5. 指导用户保存一个岗位并检查扩展弹窗中的数量。
6. 指导用户导出 `.jobs.json` 并上传；收到文件后立即进入导入流程，不再询问是否继续。

每一步只问一个确认，不要一次让用户回答多个问题。
