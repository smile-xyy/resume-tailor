# Resume-Tailor v0.4 Implementation Report

## Scope

Implemented the generalization requirements in `docs/requirements-v0.4.md`.
The release removes candidate-specific generation assumptions while preserving
the local pipeline and v0.3 output contract.

## Delivered

- Added deterministic `candidate-profile.json` generation.
- Added stable evidence IDs for identity, education, skills, and experiences.
- Replaced fixed candidate and project text with runtime evidence extraction.
- Replaced fixed project diagrams with evidence-linked experience diagrams.
- Made summary, opener, analysis, strategy, and questions candidate-dependent.
- Reordered experience blocks generically and only within their source section.
- Added evidence references to generated candidate claims.
- Raised the scoring version to `3.0.0` to invalidate incompatible caches.
- Updated the six Skill contracts and data schema.
